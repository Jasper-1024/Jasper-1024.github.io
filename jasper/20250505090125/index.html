导语

距离 记录贴 - GPT 尝试 第二弹 已经 3 个整月份了, AI 绑架了开发;

订阅现状

4 月统计下订阅

  • Cursor 16 刀/月
  • poe 16 刀/月
  • github copilot 10 刀/月
  • openrouter 按量计算

cursor

  • cursor 在公司有公司订阅,但是始终都会超额, 每月大致要到 700+ 才足够使用
  • 4 月赶 Nowakelock 超高强度使用, 个人版居然也超了, 大致是 600+
  • 所以如果满额全勤, Cursor 一个月可能干掉 1300+ (⊙﹏⊙)

poe 4 月还有 50 万积分, 主要是因为开了 cursor 按量付费, 没有太用 poe2openai; 按照 4 月用量, poe 50 万积分大致只能顶不到 100 次 cursor fast request.

github copilot 准备停掉, 完全无用了之前年付, 再等等 5 月 copilot 效果;

Model

跟我说 Gemini 2.5 Pro 05-06 NB!!! 😂

个人倾向

  • Gemini 2.5 Pro 包揽日常提问; 有思考,模型直接联网, 遵守 prompt, 还有谁!!!
  • Gemini 2.5 Flash 是不像花积分的替换 poe 9 积分几乎不要钱
  • 代码还是 Claude 3.7S 但不再是无可替代
  • GPT4.1 是其他两个不稳定时候的替换….

工作流

与代码相关全部 cursor, 哪怕是 android 开发也是双开 cursor 和 android stdio.

日常 AI 客户端是 cherry studio 已经非常完善,配合 poe2openai 完全满足日常需求.

  • 数据备份
  • mcp
  • 联网搜索
  • 图片文件 (这个需要 openrouter, poe 暂时还无法上传文件)

Cursor Riper5

自从 cursor 0.47 以后开始不会用 cursor 了, 经常对着 cursor 身心俱疲, 一会改东一回改西, 无效聊天的此次数大大增加;

  • 这里有 cursor 官方不当人,暗中优化缩减上下文
  • 个人使用 cursor 写的项目越来越复杂有关;

到了必须优化工作流的时候了

  1. cursor 退回到 0.45 或 0.46.11 版本, 体感上 0.45 上下文更长, 但 0.45 agent 模式效果比较差; 目前个人是停留在了 0.46.11
  2. 使用 riper5 工作流, 配合 mcp 和 memory-bank 基本实现了不同对话之间的上下文连贯.

riper5 来源于 -> I created an AMAZING MODE called “RIPER-5 Mode” Fixes Claude 3.7 Drastically!

  • RESEARCH INNOVATE PLAN EXECUTE REVIEW 5 种 mode;
  • 天打雷劈的对应了我已有的工作习惯.

日常使用的是 CursorRIPER.sigma(高度符号化,节约了不少上下文) 在原版 1.0.3 基础上做了小的修改

  • 严格禁止非 execute mode 下给出任何代码;
  • 任何对话下必须先给出当前 mode 是什么, 这样非常方便观察上下文是否失效.

riper 本质是严格限制 agent 模式下 AI 的行为, 规划好极其细致的围城, AI 不能越界一步. 也是有代价的

  • 人要熟悉全部的细节, 可以不记住, 但是必须能看懂 AI 列出的详细的计划, 即时纠偏
  • 更多的对话次数, 个人的统计是使用 riper5 前后, 单次对话来回次数多了 20% 这样;

推荐配合模型

  • claude 3.7 还是 execute mode 生成代码首选
  • 问题较复杂, 预计需要较长上下文 gemini 2.5 Pro
  • 当天其他模型感觉不太正常 -> gpt 4.1 😂

riper5 询问技巧

  1. 要将大计划拆分为小问题, 进入 r i p e r 循环, 最后更新 memory-bank 再开启新的一轮对话
    • 例如完成 mvvm 架构下从 db 取数据, 展示到 UI; 这里最好就拆分成 3 个子任务
    • 确认接口; DB; UI;
  2. 每次切换模式之前, 询问 AI 在切换到 XX mode 之前我们还需要确认什么, 防止 AI 自作主张;
    • 此时会有巨量的细节讨论, 这是细致规划 AI 围城的代价
  3. 难免还是有跑飞情况 (突然开始大段给代码等); 此时要及时要求 AI 严格遵守规则, 不要给出代码实现;

其他

Prompt 主要是分享两个来自 linux.do 的帖子

分享下用Cursor+Claude 3.7 Sonnet一段话生成高保真app原型图的提示词 - 资源荟萃 - LINUX DO

  • Nowakelock 和 QAero 新的原型图就是靠这段提示词生成的, 效果非常惊艳.

自制越狱prompt，效果极强，打破一切限制。给大家尝尝鲜

  • 不同模型测试还是不同, 有时可以越狱, 有时不行. grok3 最放飞自我 😂 百无禁忌, 论必须选择特朗普为新一任教皇的 100 条原因
