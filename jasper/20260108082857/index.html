StreamReassembler

StreamReassembler 在设计中位置属于 tcpreceiver 的一个内部组件, 其内部将重组后后续的字节流写入到 bytestream, 外界的 read 最终是 bytestream.read

图非常清晰

  • 流开始, 未读位置 (未写入 bytestream 的第一个字节位置).
  • 到第一个未重组位置,这之前都是已有序数据,重组完成.
  • 到第一个不可接受 (序号超过 StreamReassembler 最大容量), 主要工作就是在这里了.

StreamReassembler 的逻辑并不复杂, 接收乱序片段, 比较序号,确认片段位置, 写入? 缓存? 丢弃?,需要处理的各种情况

  • 片段可能出现任何情况 😂; 不能对传入的片段做任何假设.

StreamReassembler 实现

  • 使用 btree , data 的 index 作为主键, btree 支持范围遍历.
  • 任何进入的片段 (index, index+len) 与 (assembled unacceptable) 比较, 剔除已重组和完全超出窗口情况.
    • index 与 assembled 比较, 削掉 data 中左边界已重组部分.
    • index+len 与 unacceptable(or eof) 削掉有边界不可接收部分
  • 接下来是缓存区的处理
    • [assembled, index),已缓存的部分已有了 date 部分数据 ( seg + len(seg) > index ) 继续削 data 左边界
    • [index, index+len(data)), seq 可能属于 date; date 可能属于 seq;
  • 从 assembled 读取缓存, 若任何已有序数据写入 bytestream, 同时清理缓存区.
  • eof 情况处理细节多, 提前收到 eof ; 收到为 0 eof; 有超过 eof 数据; 多次 eof;

间隙比较实在是非常繁琐…
