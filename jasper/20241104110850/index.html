导语

最近终于将去年计划上的 nas && homelab 平台组起来了;

说是 homelab 有些名不副实, 实际上是一个 n100 主机, 加上 2 块硬盘组成 raid 0/1; 好在自己需求也不大, n100 其实可能都过剩了.

  • pve = truenas + k3s;
  • 一直需要运行的大约 10 个容器, 这个过程先不表,后面将会专门总结;

这些年手头攒了几个米家设备, 配合 HomeAssistant 终于将这些设备的联网功能用上了;

这一篇主要内容

  • 容器部署 HomeAssistant x86_64;
  • HA 联动米家设备, 纯局域网互动;
  • 非教程贴, 仅按照安装过程贴出验证有效的资料, 有问题请留言.

前排提醒: 容器部署 HA Supervised 并不受到官方支持, 其备份升级相当麻烦, 各类莫名其妙的问题;

  • 个人选择容器部署仅仅为了和其他容器部署应用统一起来;

初始化过程其实相当简单, 最大的问题是网络问题 (貌似 9 成问题都是网络问题).两个至今有效的参考

  • 关于 Home Assistant 的一些分享和建议
  • Docker安装Home Assistant (Hassio Supervisor)

对应的 docker-ompose.yaml

容器启动后, 在 n100 上初始化网络没有问题情况下, 大于 10min; 随后 ha 会监听在 8123 端口

米家设备接入 HA

小米杂货铺的设备接入 HA -> Xiaomi Miot For HomeAssistant

  • 超级方便, 感谢开发者!

提供有两种基本模式:

  • 账号集成: 配置小米账户, 直接走小米平台控制设备, 任何已经绑定了米家 app 的设备, 不再需要重新配置.
  • 本地集成: 通过 token 走局域网直接控制设备, 需要获取到设备的 token 和 ip 地址. 需要一个设备一个设备配置.

这里自然是选择 本地集成;

Xiaomi Miot For HomeAssistant 推荐通过 HACS 安装; HACS 可以简单理解为一个第三方的 HA 插件商店, 大大简化 HA 插件下载安装; 基本是按照 doc 一步一步来就好了.

  • hacs 安装: 添加 https://www.hacs.xyz/docs/use/download/download/#to-download-hacs
    • 这里会遇到一个错误: 'AddonManager.install' blocked from execution, system is not healthy

https://zyyme.com/docker-home-assistant-hassio-supervisor.html
hassio 文件夹下新建 jobs.json 并在里面写入 {"ignore_conditions": ["healthy"]}

  • 运行一次 get_hacs, 继续配置: https://www.hacs.xyz/docs/use/configuration/basic/#to-set-up-the-hacs-integration
  • hacs 还需要一个 github 账户,随便一个邮箱注册一个即可.

获取 Token

途径还是挺多的

插件参考: 获取 miio token

还有部分教程提供有 gateway3 获取, 个人测速没有成功;

另一个途径: Xiaomi-cloud-tokens-extractor 亲测.

  • 来源 https://www.home-assistant.io/integrations/xiaomi_miio/#retrieving-the-access-token

对于路由器玩出一个一个分区, 此时可能遇到设备无法通过本地集成, 即使 ip token 正确;

  • 这个是一个小米物联网协议的限制, 控制端必须要是同网段 [2]

这里就需要单独配置 nat 规则 或者设置不同区域的 IP 动态伪装.具体可以参考 openwrt 的防火墙设置;
