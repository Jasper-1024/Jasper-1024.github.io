导语

最近在重组所有个人的环境, 包括不限于 NAS 路由器 HA VPS Blog 等等, 搞得头都大了, 回到原点还是需要真正做到 lac 代码即配置,自动化取代任何可能重复的工作;

先从之前杂乱的脚本开始; 然后是 ansible 再然后是 Terraform 再然后是 k3s … 等等 重走运维之路???

脚本

这个脚本 setup_ssh.py 对应 VPS安全加固#SSH 章节, 将 用户名: 密码登录 改为 SSH key 登录;

  • python3 单文件; 依赖于 paramiko 和 fabric 两个库

功能

  • 用户名: 密码登录.
  • 设置 ssh key 登录, 将 登录配置存入 host 的 ~/.ssh/config 下
  • ssh key 成功设置后禁用 用户名: 密码登录.
