ByteStream

ByteStream 是一个有序输入输出, 类似管道. 提供 pop peek 等方法

c++ 使用到了 interface, 但是 go 这里只提供了固定方法, 并没有使用 interface:

  • 接口越大,抽象越小. ByteStream 后面并没有多态需求.
  • go 接口有虚拟调用开销

C 的习惯下:

  • 数组拷贝, 多次 mllock free;
  • 环形缓存区, 双指针, 容量 +1, 异常复杂的细节处理, 经典的教科书例子.

go 是 够用就好, 而 不是 " 最优化 "; 使用切片实现

  • 底层数组长度不关心; 只通过固定长度切片; 返回时候返回数据拷贝; Write 时 offset 超过 buffer 边界就触发 copy 重置 offset;
  • 粗粝, 但是有效. 写起来大量拷贝是真不习惯… 但是无拘泥于细节,完成 >> 完美.
